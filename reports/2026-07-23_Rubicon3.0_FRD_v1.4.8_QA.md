## 그룹별 관찰

- **정합성(CN)이 최대 리스크 그룹**: 231개 앵커 체계는 전반적으로 잘 관리되나, 섹션 간 상호참조(F↔H)에서 깨진 링크 2건 — 대형 문서에서 사람이 못 잡는 유형을 전수 색인이 잡음.
- **디자인 시스템 정합**: 폰트는 **전수 통과**(비승인 스케일 0건 — 모범적). 색상은 승인 13종+비승인 6종 혼재. 그림자는 4변형 중 2개만 토큰 일치.
- **상태 완결성**: p29 표본 기준 CTA 상태 정의 부재 — Common(C섹션)에 공통 상태 정의가 있는지 확인 필요(미확인 시 문서 전반 이슈로 확대).
- **인터랙션**: TBD 0건 — 모든 액션에 목적지가 있음(모범적). D2.0→E.T&C_Popup 참조도 유효.
- **시각 검증(p29)**: 앵커 위치·목업↔스펙 값 일치 양호. Mobile 목업은 앵커 미표기 관행(PC만 표기) — 허용 범위.
- **접근성**: 대비 위반이 반복 패턴(파일럿과 동일 원인) — 디자인 시스템 토큰 차원의 해결이 필요.

## 검사 한계

- **.fig 교차검증 불가**: Rubicon .fig는 신형 kiwi 스키마(미지 필드 57)로 현행 디코더가 실패 — 비승인 색 6종이 named style(의도)인지 raw 값(실수)인지 미확인. **이 실패는 디자인시스템 v1.7 수령 시에도 재발할 리스크** — 디코더 보강 또는 REST API 경로 필요.
- **정밀 판정 커버리지**: 요소 단위 정밀 검사(상태·경계값·흐름)는 D섹션 p29의 4개 요소만 수행. 나머지 227개 앵커는 결정적 전수 스캔(색·폰트·그림자·참조)만 적용 — C/F/H 섹션 정밀 검사는 후속 배치 필요.
- 짝없음 후보 다수(C7.1, C11.0, E1.5 등)는 상호참조 관행과 구분하려면 페이지별 시각 확인 필요 — 이번에는 번호 결번·미정의 참조만 확정 보고.
- 접근성 구현 품질(포커스·스크린리더)은 명세 검사 범위 밖.
